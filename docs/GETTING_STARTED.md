## 環境準備

### 必要な環境

- **OS**: Windows, macOS, Linux
- **Python**: 3.8以上
- **メモリ**: 4GB以上推奨
- **ストレージ**: 1GB以上の空き容量

### Pythonのインストール確認

```bash
python --version
# または
python3 --version
```

Python 3.8以上が表示されればOKです。

## インストール

### 1. プロジェクトのクローン

```bash
cd ~/Documents
git clone <repository-url> grass-coin-trader
cd grass-coin-trader
```

### 2. 必要パッケージのインストール

```bash
pip install -r requirements.txt
```

または個別に:

```bash
pip install requests numpy pandas pyarrow streamlit plotly torch scikit-learn statsmodels
```

### 3. インストール確認

```bash
python -c "import pandas, streamlit, plotly; print('OK')"
```

`OK`と表示されればインストール成功です。

## データ収集

### 1. 銘柄リストの確認

`symbols.txt`に対象銘柄（40銘柄）が記載されています：

```
# メジャー（時価総額TOP10）
BTC
ETH
BNB
...
```

### 2. 全銘柄のデータ取得

```bash
python src/data/minute_data_collector.py
```

**初回実行時**:
- 40銘柄すべての1分足データを取得
- 過去3000日分（約8年）
- 銘柄ごとに数分かかります
- **合計: 30-60分程度**

**2回目以降**:
- 最終取得時刻以降の差分のみ取得
- 数分で完了

**進捗表示**:
```
================================================================================
Bulk Data Collection
================================================================================
Symbols: BTC, ETH, BNB, XRP, ADA, ...
Time: 2025-11-01 10:00:00
================================================================================

[1/40] BTC
================================================================================
[INITIAL] BTC - Fetching 3000 days of data
================================================================================
   Progress: 50.0% (50000 records)
```

### 3. 特定銘柄のみ取得

```bash
# BTCとETHのみ
python src/data/minute_data_collector.py --symbols BTC,ETH

# 期間指定（30日分）
python src/data/minute_data_collector.py --symbols BTC --days 30
```

### 4. データサマリー確認

```bash
python src/data/minute_data_collector.py --summary
```

出力例:
```
================================================================================
Saved Data Summary
================================================================================
  BTC:
    Count: 432,000 records
    Period: 2023-01-01 00:00 ~ 2025-11-01 10:00 (1000 days)
  ETH:
    Count: 432,000 records
    Period: 2023-01-01 00:00 ~ 2025-11-01 10:00 (1000 days)
...
```

## 基本的な使い方

### ステップ1: 銘柄と時間足を選択

**左サイドバー**:
1. **銘柄**: BTC, ETH, SOLなどから選択
2. **時間足**: 1m, 5m, 15m, 1h, 4h, 1d から選択
3. **表示期間**: スライダーで調整（直近N件）

### ステップ2: チャート分析

**メインエリア**:

#### ローソク足チャート
- 緑: 終値 > 始値（上昇）
- 赤: 終値 < 始値（下落）
- ヒゲ: 高値・安値を表示

#### テクニカル指標
- **Bollinger Bands**: オレンジの帯（ボラティリティ）
- **SMA20/50**: 青と緑の線（移動平均）
- **RSI**: 0-100の指数（買われすぎ・売られすぎ）
- **MACD**: トレンドの方向と強さ

#### インタラクティブ機能
- **ズーム**: マウスドラッグ
- **パン**: Shiftキー + ドラッグ
- **ホバー**: 詳細情報表示
- **レンジセレクター**: 1時間、1日、1週間、全期間

### ステップ3: 統計情報確認

**統計情報パネル**:
- **現在価格**: 最新の取引価格
- **平均価格**: 表示期間の平均
- **期間最高/最安**: 表示期間の最高値・最安値
- **平均リターン**: 1日あたりの平均変動率
- **標準偏差**: ボラティリティ（リスク）
- **RSI**: 買われすぎ・売られすぎ判定

### ステップ4: 高度な分析ツール

**🔧 高度な分析ツール**セクション:
1. **予測モデル**を展開
   - ARIMA/GARCH予測
   - GRU深層学習予測

2. **統計分析**を展開
   - 複数銘柄の相関分析
   - ベータ分析

3. **テクニカル指標**を展開
   - ATR（ボラティリティ）
   - OBV（出来高）
   - ストキャスティクス

各ツールの使用例コードを参照して、実際に実行できます。

### ステップ5: ARIMA/GARCH予測（1日足のみ）

**時間足を「1d」に変更**すると、価格予測セクションが表示されます:

1. **「🔮 7日間の価格とリスクを予測」**ボタンをクリック
2. 予測計算中...（20-30秒）
3. 結果表示:
   - 7日間の予測価格
   - 95%信頼区間
   - ボラティリティ予測

## トラブルシューティング

### Q1: データ取得が進まない

**症状**: `python src/data/minute_data_collector.py`実行後、何も表示されない

**原因**: 出力バッファリング

**対処法**:
- Ctrl+Cで一度停止
- 再実行してみる
- ログが表示されるまで数秒待つ

### Q2: Streamlitが起動しない

**症状**: `streamlit: command not found`

**対処法**:
```bash
# Streamlitを再インストール
pip install --upgrade streamlit

# または
python -m streamlit run dashboard/main.py
```

### Q3: データが表示されない

**症状**: ダッシュボードで「データがありません」と表示される

**対処法**:
1. データ収集が完了しているか確認
   ```bash
   python src/data/minute_data_collector.py --summary
   ```

2. データファイルが存在するか確認
   ```bash
   # Windows
   dir src\data\timeseries\prices\*.parquet

   # macOS/Linux
   ls -lh src/data/timeseries/prices/*.parquet
   ```

3. データファイルがあれば、ダッシュボードで「🔄 全データ取得」ボタンをクリック

### Q4: 予測機能が使えない

**症状**: 予測ボタンが表示されない

**対処法**:
- 時間足を「1d（1日足）」に変更してください
- ARIMA/GARCH予測は日足データでのみ利用可能です

### Q5: メモリ不足エラー

**症状**: `MemoryError`が発生する

**対処法**:
- 表示期間を減らす（スライダーを左に）
- 不要なアプリケーションを終了
- システムメモリを増やす
