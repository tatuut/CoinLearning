# 📚 Week形式の詳細サンプル

このファイルは、実践的ツール習得教材（Week形式）の作成方法を示す詳細な記載例です。

**ベース**: `curriculum/week1_basics_v2.md`（100円チャレンジ開始）

---

## 🎯 Week形式の目的

**Who**: 実際に手を動かして100円→1000円を達成したい実践者
**What**: ツールの使い方、具体的な手順、トラブルシューティングを物語で学ぶ
**Why**: 読むだけでなく実行することで、スキルと自信を身につける

---

## 📋 Week形式の6パート構成

### Part 1: ストーリー導入（1000文字目安）

**目的**: 読者を物語に引き込み、今週の課題を明確にする

**コラムの例**:

```markdown
#### コラム: USDTとは？（ユウタの疑問）

**ミコ**: 「いい質問だ。まずUSDTを理解しないと先に進めない」

**USDT（テザー）とは**:
- **ステーブルコイン**（価格が安定した仮想通貨）
- **1 USDT = 1 USD（米ドル）** に固定されている
- 仮想通貨取引の**基軸通貨**

**ユウタ**: 「基軸通貨...？」

**ミコ**: 「例えば、お前がSHIBを買いたいとする」

\```
普通の考え:
日本円 → SHIB（直接買いたい）

現実:
❌ 日本円 → SHIB（このペアは存在しない）
✅ 日本円 → XLM → USDT → SHIB
\```

**ユウタ**: 「めんどくさっ！なんで直接買えないの？」

**ミコ**: 「仮想通貨の世界では、ほぼ全ての取引ペアが『○○/USDT』になってるんだ」

**取引ペアの例**:
- BTC/USDT（ビットコイン/テザー）
- ETH/USDT（イーサリアム/テザー）
- SHIB/USDT（柴犬コイン/テザー）
- DOGE/USDT（ドージコイン/テザー）

**ミコ**: 「USDTは『仮想通貨界の米ドル』だと思え。全ての取引の基準になる」

**ユウタ**: 「つまり...USDTがないと何も買えないってこと？」

**ミコ**: 「その通り。だからまずXLMをUSDTに交換する」

**ユウタ**: 「なるほど！じゃあUSDTは中間通貨みたいなもんだ」

**ミコ**: 「完璧だ」
```

**ポイント**:
- ✅ 初心者の疑問を先回り

- ✅ 図解（コードブロック）で視覚化

Successfully installed requests-2.31.0 pandas-2.0.3 ...

3. APIキーとシークレットをコピー
4. プロジェクトの`.env`ファイルに貼り付け

**.envファイルの例**:
\```
MEXC_API_KEY=your_api_key_here
MEXC_API_SECRET=your_api_secret_here
\```

**セキュリティ注意**:
⚠️ `.env`ファイルは絶対にGitにコミットしない
⚠️ APIキーは「Read」と「Trade」のみ許可（「Withdraw」は禁止）

→ トラブルシューティング参照

### トラブルシューティング

#### Q1: 「ModuleNotFoundError: No module named 'requests'」と出る

**原因**:
必要なライブラリがインストールされていない

**解決方法**:
\```bash
pip install requests pandas python-dotenv
\```

---

#### Q2: 「Permission denied」と出る

**原因**:
管理者権限が必要

**解決方法（Windows）**:
1. コマンドプロンプトを右クリック
2. 「管理者として実行」を選択
3. 再度コマンドを実行

**解決方法（Mac/Linux）**:
\```bash
sudo python market_scanner.py
\```

---

#### Q3: 「API Authentication Failed」と出る

**原因**:
APIキーが間違っているか、権限が不足

**解決方法**:
1. MEXCでAPIキーを再確認
2. 「Read」と「Trade」権限があるか確認
3. `.env`ファイルに正しく貼り付けたか確認
4. APIキーの前後にスペースがないか確認

## 📊 Week形式の重要ポイント

### 1. ストーリーパート（Part 1）のコツ

✅ **共感できる状況設定**
- × 「ある人が仮想通貨を始めた」
- ○ 「バイト代8万円で家賃5万円のユウタが、100円で草コインを始めた」

✅ **具体的な数字**
- 資金、日時、価格、手数料、すべて具体的に

✅ **失敗前提**
- 最初から完璧を求めない
- 失敗から学ぶプロセスを示す

### 2. ツール説明（Part 2）のコツ

✅ **Why → What → How の順**
1. なぜ必要か？（問題提起）
2. 何ができるか？（概要）
3. どう使うか？（手順）

✅ **対話形式で疑問を先回り**
- 読者が思いそうな疑問を、ユウタに代弁させる
- ミコが答えることで、自然に理解が深まる

### 3. 実践パート（Part 3-4）のコツ

✅ **失敗 → 反省 → 改善 → 成功**
- Part 3: 失敗体験（メンターの警告を無視）
- Part 4: 改善して成功

✅ **具体的な日時と数字**
- 「2025年10月18日 15:30」
- 「購入価格: $0.00002450」
- 「損益: -5.71%（約4.9円の損失）」

### 4. 実践ガイド（Part 5）のコツ

✅ **超詳細**
- 手順1, 手順2, ... と番号を振る
- 各手順で「やること」「期待される出力」「エラー時の対処」を明示

✅ **コピペで実行可能**
- コードブロックはそのまま実行できる形で

✅ **トラブルシューティング充実**
- よくあるエラー10個くらい
- OS別の説明

### 5. まとめ（Part 6）のコツ

✅ **Before/After**
- 資金、知識、心境の変化を明示

✅ **次への伏線**
- 次のWeekで何を学ぶか予告

✅ **読者へのエール**
- ユウタからのメッセージで励ます
