# 📚 カリキュラム作成ガイド

## 🎯 目的

質の高い実践的教材を段階的に作成するための標準手順書

**目標文字数**: 1つのWeekあたり10,000文字以上
**方針**: 一気に書かず、段階的に作成することで質を担保

---

## 📋 作成の5ステップ

### ステップ1: 概要と最終ゴール策定

**目的**: このWeekで何を学ぶか、何ができるようになるかを明確化

**作成内容**:
```markdown
# Week X: タイトル

## このWeekの概要
- 学ぶこと
- 達成目標
- 前提知識

## 最終ゴール
- このWeekが終わった時点でできること
- 使えるようになるツール
- 身につく知識・スキル

## 登場人物の成長
- ユウタの現状（Week開始時）
- ユウタの到達点（Week終了時）
- ミコが教えること
```

**ファイル保存**: `curriculum/weekX_<topic>_outline.md`

**チェックポイント**:
- [ ] Week全体の流れが明確か
- [ ] 前のWeekからの繋がりが自然か
- [ ] 最終ゴールが具体的か
- [ ] 使用するツールがリストアップされているか

## 💡 Tips

### コミットメッセージの書き方

```
Week X: ステップY-Z完了 - 内容の概要

【このコミットで追加した内容】
- ...
- ...

【次のステップ】
- ...
```

### 文字数の確認方法

```bash
# Linuxの場合
wc -m week2_technicals.md

# Windowsの場合
powershell -command "(Get-Content week2_technicals.md | Measure-Object -Character).Characters"
```

### Claude Codeとの協働

1. **一つのステップごとに依頼**
   - 「Week 2のステップ1をお願いします」
   - 「Part 3を埋めてください」

2. **フィードバックループ**
   - 各ステップ完了後にレビュー
   - 修正が必要なら指摘
   - OKならコミットして次へ

3. **並列作業は避ける**
   - 1つずつ順番に進める
   - コミット履歴を綺麗に保つ
