# 📚 カリキュラム作成ガイド

## 🎯 目的

質の高い教材を段階的に作成するための標準手順書

このガイドでは、2つの形式の教材作成方法を提供します：

### 形式1: Week形式（実践教材）

**目的**: ツールの使い方を実践的に学ぶ
**目標文字数**: 1つのWeekあたり10,000文字以上
**方針**: 一気に書かず、段階的に作成することで質を担保
**特徴**:
- ユウタとミコのストーリー形式
- 具体的な手順とコマンド例
- トラブルシューティング充実
- 初心者でもすぐに実践できる

### 形式2: Chapter形式（技術習得ストーリー）

**目的**: 技術の本質と数学的根拠を理解する
**目標文字数**: 1つのChapterあたり8,000-12,000文字
**方針**: 発明者の物語を通じて技術の必然性を理解
**特徴**:
- 歴史的な発明ストーリー
- 数学的導出と直感的説明の融合
- なぜその技術が生まれたかの理解
- 投資戦略の文脈での技術の位置づけ

## 📋 Week形式の作成手順（5ステップ）

### ステップ1: 概要と最終ゴール策定

**目的**: このWeekで何を学ぶか、何ができるようになるかを明確化

**作成内容**:

- このWeekが終わった時点でできること

## 登場人物の成長
- ユウタの現状（Week開始時）
- ユウタの到達点（Week終了時）

**チェックポイント**:

## 📖 Chapter形式の作成手順（6ステップ）

**適用対象**: 技術的な手法やアルゴリズムの深い理解を目的とする教材

**基本コンセプト**:
- 「まずは投資戦略ストーリーが先、技術は後」
- 技術は戦略を実行するための「武器」
- 発明者の苦悩と発見のドラマを通じて学ぶ
- 数学的厳密さと直感的理解の両立

### ステップ1: 技術と発明者のリサーチ

**目的**: その技術がなぜ生まれたのか、誰がどんな課題を解決しようとしたのかを調査

**調査内容**:
```markdown
## リサーチメモ

### 発明者プロフィール
- 名前: J. Welles Wilder Jr.
- 時代背景: 1978年、オイルショック後の市場混乱期
- 職業: 機械エンジニア → トレーダー
- 問題意識: 「買われすぎ・売られすぎ」を定量化したい

### 技術の歴史的背景
- 以前はどうしていたか: 経験と勘に頼る
- 何が課題だったか: 主観的判断でブレる
- どんな発想で解決したか: 工学の「正常範囲」概念を応用

### 数学的基礎
- 使われている数式: RS = avg_gain / avg_loss
- なぜその数式なのか: 上昇力と下降力の比率
- 正規化手法: 0-100スケールへの変換理由

### 現代での評価
- 学術的評価: 広く認められている
- 実務での使用頻度: 90%以上のトレーダーが使用
- 弱点と限界: トレンド相場では遅れる
```

**リサーチ方法**:
1. 原典論文を探す（Google Scholar, arXiv）
2. 発明者の伝記・インタビューを読む
3. 学術論文で引用数・評価を確認
4. 実務家のブログ・書籍で実践例を調査

**ファイル保存**: `docs/stories/XX_<技術名>_research.md`

**チェックポイント**:
- [ ] 発明者の人物像が具体的か
- [ ] 時代背景が明確か
- [ ] 数学的根拠が理解できたか
- [ ] 現代での位置づけが分かったか

## Scene 1: 1978年6月、ニューオリンズ - 5連敗の夜

J. Welles Wilder Jr.は、自宅の書斎で頭を抱えていた。

机の上には、手書きのトレード記録が散乱している。

```

6/3  金先物 買い → -$1,800
6/5  原油先物 売り → -$3,200
6/8  銀先物 買い → -$1,500
6/10 大豆先物 買い → -$2,100

「また負けた...」

Wilderは45歳。元は機械エンジニアだったが、5年前にトレーダーに転身した。

工学の世界では優秀だった。設計した機械は正確に動く。計算通りに。

しかし、相場は違った。

「なぜだ？なぜ勝てない？」

彼は移動平均線を使っていた。当時の標準的な手法だ。

教科書通りの手法。

しかし...

「遅すぎる。シグナルが出た時にはもう遅い」

「まるで、ブレーキの効かない車を運転しているようだ」

「機械なら、異常値を検知する仕組みがある」

「圧力が高すぎたら安全弁が作動する」

「価格が上がりすぎたら『買われすぎ』」
「下がりすぎたら『売られすぎ』」

「それを数値化できれば...」

机の引き出しから、方眼紙を取り出す。

```python
import pandas as pd
import numpy as np

def calculate_rsi(prices: pd.Series, period: int = 14) -> pd.Series:
    """
    RSIを計算

    Args:
        prices: 価格データ（Series）
        period: 期間（デフォルト14日）

    Returns:
        RSI値（0-100）
    """
    # Step 1: 価格変化を計算
    delta = prices.diff()

    # Step 2: 上昇と下降を分離
    gain = delta.where(delta > 0, 0)  # 上昇分だけ
    loss = -delta.where(delta < 0, 0)  # 下降分だけ（正の値に）

    # Step 3: 平均を計算（14日間）
    avg_gain = gain.rolling(window=period).mean()
    avg_loss = loss.rolling(window=period).mean()

    # Step 4: RS（相対力）を計算
    rs = avg_gain / avg_loss

    # Step 5: RSI（0-100）に正規化
    rsi = 100 - (100 / (1 + rs))

    return rsi
```

### 完全実装コード

**ファイル**: `tools/technical/rsi.py`

```python
"""
RSI (Relative Strength Index) 実装
J. Welles Wilder Jr., 1978
"""

import pandas as pd
import numpy as np
from typing import Tuple, Dict

def calculate_rsi(
    prices: pd.Series,
    period: int = 14
) -> pd.Series:
    """
    RSIを計算

    Args:
        prices: 終値データ
        period: 期間（デフォルト14）

    Returns:
        RSI値（0-100）
    """
    delta = prices.diff()
    gain = delta.where(delta > 0, 0)
    loss = -delta.where(delta < 0, 0)

    avg_gain = gain.rolling(window=period).mean()
    avg_loss = loss.rolling(window=period).mean()

    rs = avg_gain / avg_loss
    rsi = 100 - (100 / (1 + rs))

    return rsi

def interpret_rsi(
    rsi_value: float
) -> Dict[str, any]:
    """
    RSI値を解釈

    Args:
        rsi_value: RSI値

    Returns:
        解釈結果
    """
    if rsi_value > 70:
        return {
            'status': 'overbought',
            'signal': 'sell',
            'strength': (rsi_value - 70) / 30,  # 0-1
            'message': f'買われすぎ（RSI={rsi_value:.1f}）'
        }
    elif rsi_value < 30:
        return {
            'status': 'oversold',
            'signal': 'buy',
            'strength': (30 - rsi_value) / 30,  # 0-1
            'message': f'売られすぎ（RSI={rsi_value:.1f}）'
        }
    else:
        return {
            'status': 'neutral',
            'signal': 'hold',
            'strength': 0,
            'message': f'中立（RSI={rsi_value:.1f}）'
        }

def detect_divergence(
    prices: pd.Series,
    rsi: pd.Series,
    lookback: int = 14
) -> Dict[str, any]:
    """
    ダイバージェンス検出

    Args:
        prices: 価格データ
        rsi: RSIデータ
        lookback: 検出期間

    Returns:
        ダイバージェンス情報
    """
    recent_prices = prices.tail(lookback)
    recent_rsi = rsi.tail(lookback)

    # 価格のトレンド
    price_trend = 'up' if recent_prices.iloc[-1] > recent_prices.iloc[0] else 'down'

    # RSIのトレンド
    rsi_trend = 'up' if recent_rsi.iloc[-1] > recent_rsi.iloc[0] else 'down'

    # ダイバージェンス判定
    if price_trend == 'down' and rsi_trend == 'up':
        return {
            'type': 'bullish_divergence',
            'signal': 'buy',
            'message': '強気ダイバージェンス: 価格↓ but RSI↑'
        }
    elif price_trend == 'up' and rsi_trend == 'down':
        return {
            'type': 'bearish_divergence',
            'signal': 'sell',
            'message': '弱気ダイバージェンス: 価格↑ but RSI↓'
        }
    else:
        return {
            'type': 'none',
            'signal': 'hold',
            'message': 'ダイバージェンスなし'
        }

# 使用例
if __name__ == '__main__':
    from data.timeseries_manager import TimeSeriesManager

    manager = TimeSeriesManager()
    df = manager.load_data('BTC')

    # RSI計算
    rsi = calculate_rsi(df['close'])

    # 最新値の解釈
    latest_rsi = rsi.iloc[-1]
    interpretation = interpret_rsi(latest_rsi)

    print(f"BTC RSI: {latest_rsi:.2f}")
    print(f"状態: {interpretation['message']}")
    print(f"シグナル: {interpretation['signal']}")

    # ダイバージェンス検出
    divergence = detect_divergence(df['close'], rsi)
    print(f"ダイバージェンス: {divergence['message']}")
```

- [ ] Wilderの課題を理解
- [ ] 発想の転換点を理解

- [ ] Pythonコードを理解

- [ ] 3つの使い方を理解

### どちらから始めるべきか？

**初心者**: Week 1から始めて、Chapter 1は後で読む
**理論派**: Chapter 1を読んでから Week 1
**ほとんどの人**: Week 1 Part 1-3 → Chapter 1 → Week 1 Part 4-6（ハイブリッド）

### Claude Codeとの協働

#### Week形式での協働

1. **一つのステップごとに依頼**
   - 「Week 2のステップ1をお願いします」
   - 「Part 3を埋めてください」

2. **フィードバックループ**
   - 各ステップ完了後にレビュー
   - 修正が必要なら指摘
   - OKならコミットして次へ

3. **並列作業は避ける**
   - 1つずつ順番に進める
   - コミット履歴を綺麗に保つ

#### Chapter形式での協働

##### 1. リサーチフェーズでのClaude Code活用

**依頼例**:
```
「RSI（Relative Strength Index）について以下を調査してください：
1. 発明者J. Welles Wilder Jr.のプロフィールと時代背景
2. なぜRSIを発明したのか（当時の課題）
3. RSIの数学的根拠（なぜその数式なのか）
4. 現代での評価と使用頻度
5. 弱点と限界

WebSearchとWebFetchを使って、信頼できる情報源から調査してください。
結果はdocs/stories/02_rsi_research.mdに保存してください。」
```

**Claude Codeができること**:
- WebSearchで学術論文や業界記事を検索
- WebFetchで原典にアクセス
- 複数の情報源をクロスチェック
- 構造化されたリサーチメモを作成

**注意点**:
- リサーチ結果は必ず自分でレビューする
- 数学的根拠は特に慎重に確認
- 複数の情報源で裏を取る

##### 2. ストーリー執筆フェーズでのClaude Code活用

**依頼例（Scene 1）**:
```
「Chapter 2のScene 1を書いてください。

【背景情報】
- 時代: 1978年6月、ニューオリンズ
- 人物: J. Welles Wilder Jr.、45歳、元機械エンジニア
- 状況: トレードで5連敗中
- 課題: 移動平均線が遅すぎる

【書くべき内容】
1. Wilderが書斎で頭を抱えるシーン
2. トレード記録（具体的な日付と損失金額）
3. エンジニア時代の成功体験との対比
4. 「移動平均線が遅すぎる」という気づき
5. 「機械の正常範囲」という発想への伏線

【文体】
- 感情描写を豊かに
- 具体的な数字や日付を入れる
- 短い文と長い文を織り交ぜる
- 読者がWilderに感情移入できるように

【文字数】: 1,500-2,000文字」
```

**Claude Codeができること**:
- ドラマチックなストーリー展開を作成
- 感情描写と具体性のバランス調整
- リサーチ情報を自然にストーリーに織り込む
- 適切な文字数で執筆

**注意点**:
- 歴史的事実は正確に（創作と事実を区別）
- 数学的導出部分は正確性最優先
- キャラクター（Wilder、ミコ、ユウタ）の一貫性

##### 3. 数学的説明フェーズでのClaude Code活用

**依頼例**:
```
「RSIの数学的導出を、初心者にも分かるように段階的に説明してください。

【前提】
- 読者: プログラミング初心者、数学は高校レベル
- 目標: なぜその数式なのか直感的に理解できる

【説明すべきステップ】
Step 1: 価格変化を上昇と下降に分離する理由
Step 2: なぜ14日間の平均を取るのか
Step 3: なぜ比率（RS）を計算するのか
Step 4: なぜ0-100に正規化するのか

【各ステップで含めるべき内容】
- 数式
- 数式の意味（直感的説明）
- 具体例（数値を使った計算例）
- なぜこの処理が必要なのか

【禁止事項】
- 専門用語の乱用
- 証明なしの「これは正しい」という断定
- 読者を置いていくような飛躍」
```

**Claude Codeができること**:
- 数学的厳密さと直感的理解の両立
- 段階的な説明（Step by Step）
- 具体的な数値例の生成
- LaTeX数式をMarkdownで表現

**注意点**:
- 数式の正確性を必ず確認
- 「なぜ」を説明し忘れない
- 具体例は実際に計算して確認

##### 4. コード実装フェーズでのClaude Code活用

**依頼例**:
```
「RSIの完全な実装コードを書いてください。

【仕様】
- 言語: Python 3.8+
- ライブラリ: pandas, numpy
- 関数:
  1. calculate_rsi(prices, period=14): RSI計算
  2. interpret_rsi(rsi_value): RSI値の解釈
  3. detect_divergence(prices, rsi, lookback=14): ダイバージェンス検出

【要件】
- 型ヒント必須
- Docstring必須（Args, Returns, Examples）
- エラーハンドリング
- 使用例（if __name__ == '__main__':）

【コメント】
- 各ステップの意味を説明
- なぜその処理をするのか明記
- Wilderの発想との繋がりを示す

【ファイル保存先】
tools/technical/rsi.py」
```

**Claude Codeができること**:
- PEP8準拠のコード作成
- 適切な型ヒントとDocstring
- 実用的なエラーハンドリング
- テストコードの生成

**注意点**:
- 実際に動かして確認
- バックテストで検証
- エッジケース（NaN、ゼロ除算）の処理確認

##### 5. Week形式とChapter形式の並行作成

**戦略的な依頼順序**:

```
Phase 1: リサーチ（両方の基礎）
→ 「RSIについてリサーチしてください」

Phase 2: Chapter形式のストーリー執筆
→ 「Chapter 2のScene 1-5を書いてください」

Phase 3: コード実装（両方で使用）
→ 「tools/technical/rsi.pyを実装してください」

Phase 4: Week形式の実践パート
→ 「Week 2 Part 2（RSI実践）を書いてください」
→ Chapter 2のコードを参照
→ Chapter 2へのリンクを含める

Phase 5: クロスリファレンス追加
→ Chapter 2のScene 12にWeek 2へのリンク追加
→ Week 2のPart 2にChapter 2へのリンク追加
```

**メリット**:
- コードを一度書けば両方で使える
- 理論と実践が自然に繋がる
- 学習者が好きな順序で学べる

##### 6. 品質チェックでのClaude Code活用

【チェック項目】

5. コードの実用性

【出力形式】

**Claude Codeができること**:
