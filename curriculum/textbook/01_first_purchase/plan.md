## 📊 データ構造

### ツール使用の流れ
```
1. 気になる銘柄を見つける（SNS、ランキング）
   ↓
2. ダッシュボードで材料を集める
   ↓
3. Week 0の知識で分析（RSI、MACD、BB）
   ↓
4. ルールを決める（利確、損切り）
   ↓
5. MEXCで購入
   ↓
6. データ保存（詳細データ収集）
   ↓
7. 監視と決済
   ↓
8. 振り返り分析
```

### 実際のデータ例
```
SHIB（失敗例）:
- RSI: 75.2（買われすぎ）
- MACD: マイナス（デッドクロス）
- 1日変動: ±6-8%（激しい）
- 結果: -12円

DOGE（成功例）:
- RSI: 45.3（中立）
- MACD: プラス（ゴールデンクロス）
- 1日変動: ±1-3%（穏やか）
- 結果: +16円
```

## 🔑 重要コンセプト

### 1. データ駆動の意思決定
- 「かわいいから」ではなく「データで判断」
- ツールで材料を集める
- 理由を言語化する

### 2. ルールの重要性
- 買う前に決める（利確・損切り）
- 感情に流されない
- 機械的に実行

### 3. 失敗からの学習
- 100円だから失敗できる
- データで振り返る
- 次に活かす

### 4. Week 0の知識活用
- RSI 70以上は買われすぎ
- MACDゴールデンクロスは買いサイン
- ボリンジャーバンド上限は過熱

## 📝 教育的工夫

### 反芻パターン
```
1回目: ミコが説明
2回目: ユウタが実践で確認
3回目: 失敗して振り返り
4回目: 成功して理解

→ 4段階で定着
```

```

- 激しい値動き vs 穏やかな値動き

→ 違いを明確に
```
