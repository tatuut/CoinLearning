## 📖 各ファイルの役割

### 🗺️ [LEARNING_PATH.md](./LEARNING_PATH.md) - 学習全体の地図

**何をするファイル**: 学習の進め方を3階層で定義

```
Level 0: 抽象的大目標「楽して生きる」
    ↓
Level 1: 具体的目標「100円 → 1000円 → 10000円」
    ↓
Level 2-A: 概念チェックリスト（順不同、動的追加）
Level 2-B: Week形式の順番（Week 0, 1, 1.5, 2...）
```

**いつ読む**:
- カリキュラムを始める前に全体像を把握
- 現在地を確認したいとき
- 次に何を学ぶか迷ったとき

**リンク**: [LEARNING_PATH.md](./LEARNING_PATH.md)

---

### 🌳 [CONCEPT_TREE.md](./CONCEPT_TREE.md) - 概念チェックリスト

**何をするファイル**: 習得すべき概念をTree構造で管理

```
📊 分析方法
  ├── トレンド系（EMA, MA, ICHIMOKU...）
  ├── オシレーター系（RSI, MACD, KDJ...）
  └── ボラティリティ系（BOLL, ATR...）
🎯 取引スキル
🧠 心理・マインド
📈 戦略
```

**特徴**:
- チェックボックスで進捗管理
- 順不同（必要なものから学ぶ）
- 動的に追加可能
- 習得率の可視化

**いつ更新する**:
- 新しい概念を学んだとき（チェック）
- 実践で新しい概念が必要になったとき（追加）

**リンク**: [CONCEPT_TREE.md](./CONCEPT_TREE.md)

## 🎯 カリキュラムの使い方

### 1. 初めての方

```
1. このREADME.md を読む（今ここ）
    ↓
2. LEARNING_PATH.md で全体像を把握
    ↓
3. textbook/00_foundations.md で基礎知識を学ぶ
    ↓
4. CONCEPT_TREE.md で習得状況をチェック
```

### 2. 学習中の方

```
1. LEARNING_PATH.md で現在地を確認
    ↓
2. 次のWeekの教材を読む（textbook/）
    ↓
3. 実践で試す
    ↓
4. CONCEPT_TREE.md にチェック ✅
```

### 3. 復習したい方

```
1. CONCEPT_TREE.md で未習得項目を確認
    ↓
2. 該当するWeekの教材を復習（textbook/）
    ↓
3. 理解できたらチェック ✅
```

## 🔄 学習の流れ

### 基本サイクル

```
📖 理論を学ぶ（textbook/）
    ↓
💻 実践する（実際に取引）
    ↓
📊 振り返る（ダッシュボード）
    ↓
🎯 新しい課題を発見
    ↓
📚 新しい概念を学ぶ（動的に追加）
    ↓
📖 に戻る
```

### 動的な学習

Week 1.5以降は、**実践で必要になった指標・概念を学びます**。

```
実践中に遭遇
    ↓
「この状況、今の知識じゃ分からない！」
    ↓
新しい指標・概念を学ぶ
    ↓
新しいWeekまたはコラムを追加
    ↓
CONCEPT_TREE.mdにチェック ✅
```

## 💡 Tips

### カリキュラムの特徴

1. **動的な学習パス**: 固定された順番ではなく、実践で必要なものから学ぶ
2. **3階層の目標管理**: 抽象的大目標 → 具体的目標 → 小目標
3. **Tree構造の概念管理**: 順不同で習得状況を管理
4. **実践と理論の統合**: Week 1.5以降はユウタと一緒に実践

### よくある質問

**Q**: Week 1の次は必ずWeek 1.5をやらないといけない？
**A**: いいえ。あなたの実践で必要になったものから学んでください。

**Q**: CONCEPT_TREEの概念を全部習得しないといけない？
**A**: いいえ。実践で必要なものだけで十分です。

**Q**: 新しい指標を学びたいときはどうすればいい？
**A**: CONCEPT_TREEの候補リストを見て、実践で試してみてください。
